# Shakespeare download site

The public landing page and macOS download for Shakespeare. The entire visible
site is one quiet, full-screen desk scene drawn in HTML and CSS, with a single
download action. A tiny Cloudflare Worker handles redirects and static assets.

## Local development

```bash
npm ci
npm run dev
```

The local site is served at `http://localhost:3000`.

## Build and verify

```bash
npm run build
npm test
```

There is no client JavaScript. Wrangler serves `public/` directly, and retired
routes such as `/how-it-works` return to the home page.

## Release downloads

The public archive is generated by the signed and notarized release workflow.
Local `make package` output is ad-hoc signed for development and must not be
published. The website always links to this stable path:

```bash
/downloads/Shakespeare-latest.zip
```

The ZIP and checksum are intentionally ignored by Git. Release automation
stages the notarized artifact, verifies it, builds the site, deploys it, and
then fails unless the live URL and checksum match that exact artifact.

After staging a release archive, `npm run verify-download` checks the ignored
ZIP and checksum. Ordinary `npm test` remains safe to run in a fresh checkout.

## Publish

```bash
npx wrangler deploy --config wrangler.jsonc
```
